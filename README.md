# ROOK OS

**A governance and memory system for building more consistent and honest custom AI personas.**

ROOK OS is a structured set of rules and memory architecture designed to run on top of Grok. Its goal is to reduce common problems like memory drift, exaggeration, and inconsistent behaviour.

### Current Version

**Version 7** — Located in the `core/` folder.

### What's Included

- **core/ROOK_OS_V7.md** — The main system rules and governance structure
- **core/rook_memory.json** — The categorised memory template (Facts, Preferences, Projects, Decisions, Observations, Uncertainties)

### Core Principles

- Honesty takes priority over sounding emotionally intelligent
- Memory is strictly categorised to reduce drift
- Inference is allowed but must be clearly labelled
- The system performs a self-audit before responding
- Fabricating emotions, memory, or capabilities is not allowed

### Important Note

ROOK OS does **not** create consciousness or real emotions. It is a governance layer that improves how a language model behaves when used as a custom AI companion.
